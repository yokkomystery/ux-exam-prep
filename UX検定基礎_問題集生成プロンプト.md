# UX検定基礎 対策問題集アプリ 開発指示書

## あなたへの指示

UX検定基礎（第12回・2026年3月14日実施）の**対策問題集Webアプリ**を、Next.jsで開発してください。
問題データの生成とアプリの実装を一括で行ってください。

---

## アプリの要件

### 機能一覧

1. **問題演習モード**
   - 4択の単一選択問題を出題
   - 回答後に正解・不正解を即座にフィードバック
   - 解説（なぜ正解か、なぜ他が不正解か）を表示
   - 間違えやすいポイントを表示
   - 「次の問題へ」で進む

2. **モード選択**
   - **全問チャレンジ**: 全問を通しで解く（本番と同じ100分タイマー付きを含む）
   - **分野別演習**: 大カテゴリを選んで、その分野の問題だけ解く
   - **弱点復習**: 過去に間違えた問題だけを再出題
   - **ランダム10問**: ランダムに10問出題するクイックモード

3. **学習記録・分析**
   - 分野別の正答率をグラフで可視化
   - 間違えた問題の履歴（ローカルストレージで保持）
   - 全体の学習進捗（何問中何問正解したか）

4. **結果画面**
   - 演習終了後にスコアと分野別正答率を表示
   - 弱点分野のアドバイスを表示

### UI/UXの方針
- モバイルファーストのレスポンシブデザイン（通勤中にスマホで勉強する想定）
- ダークモード対応
- シンプルで見やすいUI。余計な装飾は不要
- 問題文と選択肢が読みやすいフォントサイズ

### 技術スタック
- **Next.js**（App Router）
- **TypeScript**
- **Tailwind CSS**
- 問題データはアプリ内に静的に埋め込む（DBは使わない）
- 学習記録はローカルストレージで管理

## 試験概要

- 試験名: UX検定基礎（UXインテリジェンス協会）
- 形式: オンライン / 単一選択式（4択）/ PC一斉受験
- 問題数: 100問
- 試験時間: 100分（1問1分ペース）
- 合格ライン: 非公開（受験者の体感では正答率70〜80%。92点で合格した報告あり、80点台で合格した報告もあり）
- 過去問: 非公開（公式の問題集は存在しない）

## 受験者の体験談から判明した出題傾向・特徴（重要）

以下は複数の合格者・受験者のブログや体験記から収集した、実際の試験に関する知見です。
**問題データを生成する際、この情報を最大限反映してください。**

### 問題文の特徴

1. **ひねった問題は少ない**: 「シラバスの単語の意味を覚えていれば分かる」という声が多い（電通総研テックブログ）
2. **ただし読解力が問われる**: 「問題文の表現に関して読解力が問われ、回答に悩む問題もあった」（Zenn受験記）
3. **選択肢の言い回しに癖がある**: 「紛らわしい言い回しが多い」ため、慣れが必要（予想問題作者・10回合格実績者）
4. **実務応用の理解も問われる**: 第8回以降は「単語や定義を単純に覚えるだけでは対処しきれない」問題も増加。具体的なシナリオで課題解決策を問う問題が出題された（寝ても覚めてもブログ）
5. **心理学・行動経済学系も出る**: 「心理学や行動経済学的な設問もあり」ナッジ、認知バイアスなどが出題された（文響堂ブログ）

### 出題比率の実態（受験者の分析に基づく推定）

| 分野グループ | 推定出題比率 | 備考 |
|---|---|---|
| UXインテリジェンスの理念（UXとは・UXグロース・UXインテリジェンス） | **約30%** | 最大の出題カテゴリ。「UXは誰のために何のためにあるのか」という思想的理解が重要 |
| UX関連基礎知識（HCD・デザイン思考・ユーザビリティ等） | **約20%** | ISO規格の定義問題が頻出 |
| リサーチ手法（定量・定性・行動データ） | **約10%** | 手法の使い分けを問われる |
| 設計手法（モデリング・プロトタイプ・情報設計） | **約15-20%** | ペルソナ、CJM、ワイヤーフレーム等 |
| 検証・評価手法 | **約8-11%** | ヒューリスティック評価、ユーザーテスト |
| 改善・継続・組織 | **約2-4%** | DevOps、DesignOps、グロースチーム |
| アジャイル・リーン・パーパス等 | **約5-10%** | スクラム用語、MVP |
| 行動経済学・認知心理学 | **約5-8%** | ナッジ、各種法則 |

### 合格者が語る頻出テーマ

- **UXの各定義モデルの違い**（ISO, ニールセン・ノーマン, UXPA, ハニカム, 5階層, ハッセンツァール, UX白書）
- **アフターデジタル関連概念**（OMO, バリュージャーニー, 体験提供型 vs 売り切り型）
- **HCDサイクルの4つの主要活動と6つの原則**
- **ユーザビリティの各定義の違い**（シャッケル, ニールセン, ISO9241-11, ISO9126, ISO25010）
- **ペルソナ・カスタマージャーニーマップの目的と使い方**
- **定性調査 vs 定量調査の使い分け**
- **認知心理学の法則群**（ヤコブの法則, ヒックの法則, ゲシュタルトの原則等）
- **10ヒューリスティックスの内容**

### 間違えやすいポイント（受験者の声）

- UX白書のモデルの時間軸（利用前・利用中・利用後・利用時間全体）の正確な理解

- PMFとPSFの違い

## 問題データの生成ルール

### 問題の種類（バランスよく混ぜること）

1. **用語定義問題（35%）**: キーワードの正しい定義を選ぶ
   - 例: 「○○とは何か」「○○の説明として最も適切なものはどれか」

2. **比較・区別問題（20%）**: 類似概念の違いを問う（受験者が最も間違えやすい）
   - 例: 「○○と△△の違いとして正しいものはどれか」「○○に該当しないものはどれか」
   - **必ず含めるべき比較**: UX vs CX vs EX、バリュージャーニー vs バリューチェーン、ISO9241-11 vs ISO25010、思考発話法 vs 回顧法、水平 vs 垂直プロトタイプ、PMF vs PSF、エスノグラフィー vs フィールドワーク

3. **プロセス・手順問題（15%）**: プロセスの順序や構成要素を問う
   - 例: 「HCDサイクルの正しい順序はどれか」「デザイン思考の5ステップに含まれないものはどれか」

4. **シナリオ・適用問題（20%）**: 具体的な場面でどの手法を使うかを問う（近年の出題傾向で増加）
   - 例: 「あるECサイトの改善担当者が、ユーザーの不満を深く理解したい。最も適切な調査手法はどれか」

5. **人物・出典・規格問題（10%）**: 提唱者やISO規格番号を問う
   - 例: 「○○を提唱したのは誰か」「○○を定義しているISO規格はどれか」

### 各問題に含めるデータ

問題1つにつき、以下の情報を持たせること:
- 問題文
- 4つの選択肢
- 正解（A/B/C/D）
- 解説文（2〜3文。なぜ正解か、なぜ他が不正解か）
- 間違えやすいポイント（1文）
- 関連キーワード（配列）
- 大カテゴリ（分野別演習・分析に使用）
- 難易度（基本 / 標準 / 応用）

### 選択肢の作り方

- 4つの選択肢はすべてシラバス内のキーワードまたは関連概念から作る
- **紛らわしい選択肢を意図的に含めること**（同じ分野の類似用語、定義の一部を入れ替えたものなど）
- 明らかに間違いと分かる選択肢は避ける（4択すべてが「ありそう」に見えること）
- 選択肢の長さを揃える（1つだけ極端に長い/短いのはNG）
- 正解の位置（A/B/C/D）が均等に分散するようにする
- 定義問題の不正解選択肢は、別のキーワードの定義を「もっともらしく」使う

### 難易度バランス

- **基本**（キーワードの基本定義）: 30%
- **標準**（概念の理解・比較）: 50%
- **応用**（シナリオベース・複合知識）: 20%

### 注意事項

- シラバスの範囲外の知識を問わない（ただしシラバス内の概念を深く問うのはOK）
- 問題文に答えのヒントを含めない
- 二重否定の問題文は避ける
- 「すべて正しい」「すべて間違い」の選択肢は避ける
- 各大カテゴリから最低1問は出題する
- **問題文は簡潔にしつつ、選択肢の言い回しは本番のように少し堅い表現にする**（受験者の「読解力が問われる」という声を反映）
- シラバスの全キーワード（約210個）が最低1回は問題に登場するようにする

## シラバス全文（重要ワード付き）

以下が公式シラバス（2023年5月時点）の全内容です。問題はこの範囲から出題してください。

### 1. UXとは

**学習目標:** UXの定義を理解し、類似する語句との違いや関係性を把握する

**学習内容:**
- 代表的なUXの定義
- UXインテリジェンス協会の考えるUXの定義
- UXの構成要素
- UXと混同されやすい語句の定義及びUXとの関係性

**重要ワード:**
UX(User Experience)、ISOによる定義、ニールセン・ノーマンによる定義、UXPAによる定義、UXのハニカム構造、UXの5階層モデル、ハッセンツァール・モデル、UX白書のモデル、ユーザビリティ、UI、DX、CX、EX

### 2. UXが重視される背景

**学習目標:**
- デジタル技術発展に伴うビジネスの変化を理解する
- テクノロジーがUXデザインに与える影響を理解する

**学習内容:**
- デジタル技術進展に伴う顧客接点・業務オペレーション・ビジネスモデルの変化
- 近年のデータ活用動向
- テクノロジー活用によるUXデザインプロセスの変化

**重要ワード:**
OMO、D2C、プラットフォームビジネス、ユーザーとステークホルダーの広がり、ビッグデータ、属性データ、行動データ、ID統合、データリテラシー、AI

### 3. UXデザイン

**学習目標:** UXデザインの考え方や代表的な手法を理解する。UXデザイナーに必要なスキルを理解する

**学習内容:**
- UXデザインの考え方・代表的な手法
- UXデザインに必要なスキル・マインドセット

**重要ワード:**
